# LiLink 测试与运维备忘

## 一、本地开发（界面 + 种子数据）

### 前置条件

- Node.js（版本以仓库 `engines` 为准）
- Docker Desktop

### 一次性环境配置

- 将 `apps/api/.env.example` 复制为 `apps/api/.env`（可选：在仓库根目录再建 `.env` 放共用密钥；`scripts/load-env.mjs` 会先读根目录，再由 `apps/api/.env` 覆盖）。
- `DATABASE_URL` 指向 `docker-compose.yml` 中的本地 Postgres（示例默认：`postgresql://lilink:lilink@127.0.0.1:5432/lilink?schema=public`）。
- `JWT_SECRET`、`ADMIN_JWT_SECRET`、`CRON_SECRET` 须为**互不相同的字符串**，且每个**至少 16 个字符**。
- 设置 `ADMIN_BOOTSTRAP_EMAIL` 与 `ADMIN_BOOTSTRAP_PASSWORD`（密码**至少 12 个字符**），用于创建管理员；访问 `/admin` 时使用相同账号。
- `SMTP_HOST=127.0.0.1`、`SMTP_PORT=1025` 对应 `docker compose up` 中的 **Mailpit**（Web：http://localhost:8025）。

### 常用命令（仓库根目录）

```bash
npm install
npm run infra:up
npm run db:generate
npm run db:bootstrap
npm run db:seed
```

创建首个管理员（从 `.env` 读取 `ADMIN_BOOTSTRAP_*`）：

```bash
cd apps/api && node scripts/bootstrap-admin.mjs && cd ../..
```

启动 Web、API 与联调监听：

```bash
npm run dev
```

Postgres 已启动且迁移已应用时，在仓库根目录执行：`npm run test --workspace api`。

## 二、云端（生产部署与运维）

以下默认服务器路径为 `/home/admin/lilink`，容器与文档一致时使用 `docker-compose.prod.yml`。

### 部署与重启

```bash
cd /home/admin/lilink

# 重建并启动
docker compose -f docker-compose.prod.yml build api
docker compose -f docker-compose.prod.yml up -d

# 仅重启 API
docker compose -f docker-compose.prod.yml up -d --force-recreate api

# 日志与健康检查
docker logs --tail 100 lilink-api
curl -I https://api.lilink.top/v1/health

# Caddy
sudo systemctl reload caddy
```

### 数据库

```bash
# 执行迁移（在 api 容器内）
docker exec lilink-api npx prisma migrate deploy

# 默认种子（学校 + 问卷 + 轮次，无用户）
docker exec lilink-api node scripts/seed-defaults.mjs

# 全量种子（含测试用户，仅开发/演练环境；若在服务器上需在有源码与依赖的环境中执行）
# npm run db:seed

# 手动备份
bash scripts/backup-postgres.sh

# 从备份恢复
gunzip -c /home/admin/backups/lilink/<file>.sql.gz \
  | docker exec -i lilink-postgres psql -U lilink -d lilink
```

定时备份：北京时间每日3:00，保留 7 天，目录 `/home/admin/backups/lilink/`。

### 生产栈验收流程（`lilink-api` 容器）

1. `docker compose -f docker-compose.prod.yml up -d`
2. `docker exec lilink-api npx prisma migrate deploy`
3. `docker exec lilink-api node scripts/seed-defaults.mjs`（或按环境使用全量种子，见上节）
4. 打开 `/admin`，进入 **轮次**，找到状态为 OPEN 的轮次
5. 点击 **强制执行**
6. 分别以 Alice、Bob、Carol 登录 `/dashboard` 核对匹配结果
