# LiLink Operations Cheat Sheet

## 1. Production Deploy / Restart

```bash
cd /home/admin/lilink

# Rebuild & start
docker compose -f docker-compose.prod.yml build api
docker compose -f docker-compose.prod.yml up -d

# Restart API only
docker compose -f docker-compose.prod.yml up -d --force-recreate api

# Logs & health
docker logs --tail 100 lilink-api
curl -I https://api.lilink.top/v1/health

# Caddy
sudo systemctl reload caddy
```

## 2. Database

```bash
# Run migrations (inside container)
docker exec lilink-api npx prisma migrate deploy

# Seed defaults (schools + questionnaire + cycle, no users)
docker exec lilink-api node scripts/seed-defaults.mjs

# Full seed (includes test users, dev only)
npm run db:seed

# Manual backup
bash scripts/backup-postgres.sh

# Restore from backup
gunzip -c /home/admin/backups/lilink/<file>.sql.gz \
  | docker exec -i lilink-postgres psql -U lilink -d lilink
```

Auto-backup: daily 3:00 AM (Beijing), keeps 7 days. Files in `/home/admin/backups/lilink/`.

## 4. Test Users Management

In admin panel (`/admin/users`):
- **Mark**: click user → "标记为测试" button
- **Filter**: tabs "全部用户 / 真实用户 / 测试用户"
- **Bulk delete**: switch to "测试用户" tab → "删除全部测试用户"

## 5. Admin Capabilities

| Feature | Entry |
|---------|-------|
| Capacity limit (registration cap) | `/admin` → 灰度控制 |
| Edit user info (name, email, profile) | `/admin/users` → select user → 编辑资料 |
| Change user status | `/admin/users` → select user → status buttons |
| Merge schools | `/admin/schools` → ⇄ on source → "合并到此" on target |
| Run match cycle | `/admin/cycles` → select cycle → 强制执行 |
