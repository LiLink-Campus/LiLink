## 2. 问卷说明

当前种子会同步一套 24 题关系匹配问卷。问卷分成两部分：

- 硬条件题：出生日期、希望对方年龄范围、性别、希望对方性别、颜值、人种等
- 软问卷题：关系预期、推进节奏、沟通方式、修复方式、价值观、支持需求、约会偏好等

### 2.1 多选题上限

现在每一道多选题都可以单独配置“最多可选几个”，前端和后端都会同时校验。

当前种子里默认是：

- `values`：最多选 `4`
- 其他软问卷多选题：最多选 `3`

如果用户超过上限：

- 前端会阻止继续勾选
- 后端保存时也会拒绝超选数据

## 3. 本地测试流程

1. 执行 `docker compose up -d`
2. 如果数据库为空，执行 `npm run db:bootstrap`
3. 执行 `npm run db:seed`
4. 打开管理员后台 `/admin`
5. 进入「轮次」，找到当前 `OPEN` 的轮次
6. 点击「强制执行」，不必等待真实 reveal 时间
7. 分别用 Alice、Bob、Carol 登录 `/dashboard` 查看结果

## 4. 这次种子后的重点检查项

- 后台「问卷」页是否出现 24 道新题
- 多选题是否显示并保存各自的“最多可选”设置
- Alice / Bob 是否能看到匹配成功页
- Carol 是否继续表现为“本轮无匹配对象”
- 批量账号里是否仍能覆盖“未提交 / 草稿 / 已提交”等列表筛选场景

## 5. 补充说明

- 重新执行 `db:seed` 会同步当前问卷版本和测试账号答案，不适合直接在生产环境随意跑
- 如果某轮次已经被真实使用，重新 seed 会刷新当前开放轮次的时间和状态
- 每次 `db:seed` 结束后，终端仍会打印密码和测试账号摘要
