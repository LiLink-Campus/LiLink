## 2. 问卷说明

当前种子会同步一套 24 题关系匹配问卷。问卷分成两部分：

- 硬条件题：出生日期、希望对方年龄范围、性别、希望对方性别、颜值、身高（及希望对方身高范围）等
- 软问卷题：关系预期、推进节奏、沟通方式、修复方式、价值观、支持需求、约会偏好等

### 2.1 多选题上限

现在每一道多选题都可以单独配置"最多可选几个"，前端和后端都会同时校验。

当前种子里默认是：

- `values`：最多选 `4`
- 其他软问卷多选题：最多选 `3`

如果用户超过上限：

- 前端会阻止继续勾选
- 后端保存时也会拒绝超选数据

## 6. 生产后端部署 / 重启

生产环境当前只在服务器上运行后端、数据库和宿主机 Caddy。

常用命令：

- 首次部署或代码更新后重建后端镜像：`docker compose -f docker-compose.prod.yml build api`
- 启动或拉起后端服务：`docker compose -f docker-compose.prod.yml up -d`
- 只重启后端 API：`docker compose -f docker-compose.prod.yml up -d --force-recreate api`
- 查看后端状态：`docker compose -f docker-compose.prod.yml ps`
- 查看后端日志：`docker logs --tail 100 lilink-api`
- 重新加载宿主机 Caddy：`sudo systemctl reload caddy`
- 检查健康接口：`curl -I https://api.lilink.top/v1/health`

服务器重启后，Docker 和 Caddy 会自动拉起；如果需要手动确认，可按下面顺序执行：

1. 进入项目目录：`cd /home/admin/lilink`
2. 拉起后端服务：`docker compose -f docker-compose.prod.yml up -d`
3. 重新加载 Caddy：`sudo systemctl reload caddy`
4. 检查接口：`curl -I https://api.lilink.top/v1/health`

## 9. 仅种子默认仓库数据（不含用户）

生产环境如果只需要初始化学校、问卷和匹配轮次（不创建任何测试用户），使用 `seed-defaults` 脚本。

### 9.1 脚本说明

`apps/api/scripts/seed-defaults.mjs` 会执行以下操作：

- 写入 9 所学校（含 17 个邮箱域名）
- 同步 24 道关系匹配问卷（与 `seed.ts` 完全一致）
- 创建或更新一个 `OPEN` 状态的匹配轮次（reveal 时间为执行时刻 +7 天）
- **不创建任何用户账号**

### 9.2 在生产服务器上执行

由于 PostgreSQL 只在 Docker 网络内可访问，需要通过 API 容器执行：

```bash
# 方式一：直接在容器内执行（脚本已随镜像部署）
docker exec lilink-api node scripts/seed-defaults.mjs

# 方式二：如果镜像尚未包含最新脚本，先拷贝再执行
docker cp apps/api/scripts/seed-defaults.mjs lilink-api:/app/scripts/seed-defaults.mjs
docker exec lilink-api node scripts/seed-defaults.mjs
```

### 9.3 在本地开发环境执行

```bash
# 需要 DATABASE_URL 环境变量可用
npm run db:seed-defaults
```
